# TraceRTM Database Migrations

This directory contains Alembic database migrations for TraceRTM.

## Migration Files

### 000_initial_schema.py
Creates the initial database schema including:
- PostgreSQL extensions (vector, pg_trgm)
- Core tables (projects, items, links, agents, agent_events)
- Indexes for performance
- Full-text search indexes
- Automatic timestamp triggers

### 001_add_change_tracking.py
Adds change tracking infrastructure:
- change_log table for tracking changes
- Triggers on items and links tables
- Indexes for efficient change processing

### 002_add_materialized_views.py
Creates materialized views for performance optimization

### 003_add_refresh_functions.py
Adds functions to refresh materialized views

### 004_add_remaining_views.py
Additional materialized views

### 005_update_refresh_all_views.py
Updates the refresh_all_views function

### 006_add_priority_owner_to_items.py
Adds priority and owner columns to items table

## Usage

### Run All Migrations
```bash
alembic upgrade head
```

### Create New Migration
```bash
alembic revision --autogenerate -m "description of changes"
```

### Rollback One Migration
```bash
alembic downgrade -1
```

### Show Current Version
```bash
alembic current
```

### Show Migration History
```bash
alembic history --verbose
```

## Database Schema Overview

### Tables

#### projects
- id (varchar 255, PK)
- name (varchar 255, unique)
- description (text)
- project_metadata (jsonb)
- created_at, updated_at (timestamptz)

#### items
- id (varchar 255, PK)
- project_id (varchar 255, FK)
- title (varchar 500)
- description (text)
- view (varchar 50) - which view this item belongs to
- item_type (varchar 50) - type within the view
- status (varchar 50) - todo, in_progress, complete, etc.
- priority (varchar 50) - low, medium, high
- owner (varchar 255) - assigned owner
- parent_id (varchar 255, FK) - hierarchical structure
- item_metadata (jsonb) - flexible metadata
- version (int) - optimistic locking
- deleted_at (timestamptz) - soft delete
- created_at, updated_at (timestamptz)

#### links
- id (varchar 255, PK)
- project_id (varchar 255, FK)
- source_item_id (varchar 255, FK)
- target_item_id (varchar 255, FK)
- link_type (varchar 50) - implements, tests, depends_on, etc.
- link_metadata (jsonb)
- created_at, updated_at (timestamptz)

#### agents
- id (varchar 255, PK)
- project_id (varchar 255, FK)
- name (varchar 255)
- agent_type (varchar 50)
- status (varchar 50)
- agent_metadata (jsonb)
- last_activity_at (varchar 50)
- created_at, updated_at (timestamptz)

#### agent_events
- id (varchar 255, PK)
- project_id (varchar 255, FK)
- agent_id (varchar 255, FK)
- event_type (varchar 50)
- item_id (varchar 255, FK, nullable)
- event_data (jsonb)
- created_at, updated_at (timestamptz)

### Extensions

#### vector
Enables pgvector support for:
- Semantic search
- Embedding storage
- Vector similarity queries

#### pg_trgm
Enables trigram matching for:
- Fuzzy text search
- Fast LIKE queries
- Similarity ranking

### Indexes

#### Full-Text Search
- idx_items_title_trgm (GIN index on items.title)
- idx_items_description_trgm (GIN index on items.description)

#### Performance Indexes
- Composite indexes on frequently queried columns
- Foreign key indexes
- View-specific indexes

## Environment Configuration

The database URL is configured in `alembic.ini`:
```ini
sqlalchemy.url = postgresql://localhost/tracertm
```

Override with environment variable:
```bash
export DATABASE_URL=postgresql://user:pass@host/dbname
# Then update alembic.ini or use programmatic configuration
```

## Best Practices

1. **Always review autogenerated migrations** - Alembic's autogenerate is helpful but not perfect
2. **Test migrations on a copy** - Test upgrade and downgrade before applying to production
3. **Keep migrations atomic** - Each migration should do one logical thing
4. **Document complex migrations** - Add comments explaining non-obvious changes
5. **Maintain upgrade/downgrade symmetry** - Ensure downgrade reverses upgrade

## Troubleshooting

### Migration conflicts
If you get conflicts between migrations:
```bash
# Show current state
alembic current

# Show pending migrations
alembic heads

# Manually resolve by editing migration files
```

### Failed migration
If a migration fails partway:
```bash
# Check current version
alembic current

# Fix the migration file
# Then try again
alembic upgrade head
```

### Reset database (development only)
```bash
# Drop all tables
alembic downgrade base

# Or drop and recreate database
dropdb tracertm
createdb tracertm
alembic upgrade head
```
