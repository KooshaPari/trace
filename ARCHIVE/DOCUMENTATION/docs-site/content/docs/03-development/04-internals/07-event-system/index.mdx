## Role in the architecture
- Emit domain events for create/update/delete.
- Use durable queues with at-least-once delivery.
- Deduplicate and idempotently apply consumers.

## Considerations
- Throughput vs. latency trade-offs.
- Failure domains and retry/backoff strategies.
- Data model ownership and boundaries.
