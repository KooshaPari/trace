## What this covers
- How to manage workflows with the Trace CLI
- Expected inputs, flags, and outputs
- Safety checks before executing

## Quick commands
```bash
trace workflows list --project <id>
```

```bash
trace workflows trigger --workflow <name> --project <id> --payload payload.json
```

```bash
trace workflows status --run <run-id>
```

## Best practices
- Use `--project` to scope actions and avoid cross-project edits.
- Prefer idempotent operations (`trace ... upsert`) in automation.
- Pipe to jq for verification before destructive changes.
