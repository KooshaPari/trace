# CLI Configuration

Configure the TraceRTM CLI to connect to your API instance and customize behavior.

## Configuration File Location

Configuration is stored in:

- **macOS/Linux**: `~/.config/tracertm/config.json`
- **Windows**: `%APPDATA%\tracertm\config.json`

## View Configuration

```bash
# Show all settings
trace config show

# Show specific setting
trace config get api-url
```

## Set Configuration

### API Endpoint

```bash
# Production
trace config set api-url https://api.tracertm.dev

# Self-hosted
trace config set api-url https://your-domain.com/api/v1
```

### API Key

```bash
trace config set api-key rtm_live_abc123...
```

### Default Project

```bash
trace config set default-project proj_123
```

### Output Format

```bash
# JSON output
trace config set output-format json

# Table output (default)
trace config set output-format table

# YAML output
trace config set output-format yaml
```

## Environment Variables

You can also use environment variables:

```bash
# Set in shell
export TRACERTM_API_URL="https://api.tracertm.dev"
export TRACERTM_API_KEY="rtm_live_abc123..."
export TRACERTM_PROJECT_ID="proj_123"

# Use CLI
trace project list
```

Environment variables override config file settings.

## Configuration Options

| Option | Description | Default |
|--------|-------------|---------|
| `api-url` | API endpoint URL | `https://api.tracertm.dev` |
| `api-key` | API key for authentication | None |
| `default-project` | Default project ID | None |
| `output-format` | Output format (json/table/yaml) | `table` |
| `timeout` | Request timeout (seconds) | `30` |
| `verbose` | Verbose output | `false` |

## Example Configuration

```json
{
  "api-url": "https://api.tracertm.dev",
  "api-key": "rtm_live_abc123...",
  "default-project": "proj_123",
  "output-format": "json",
  "timeout": 30,
  "verbose": false
}
```
