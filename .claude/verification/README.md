# Verification Directory

This directory contains quality gate verification artifacts for the TracerTM project.

## Critical Tier Requirements

TracerTM is configured as a **critical-tier** project, which enforces:

1. **Supply Chain Security** (via `qa-supply-chain-verifier.sh`)
   - SLSA provenance attestation
   - Cryptographic signatures (cosign)
   - Rekor transparency log entries

2. **Governance & Reliability** (via `qa-verifier-dispute-gate.sh`)
   - Dispute workflow documentation
   - Verifier challenge process
   - Auditable decision records

## Setup Instructions

### Quick Start (Development Mode)

To bypass supply chain verification during local development:

```bash
export QA_SUPPLY_CHAIN_FAIL_CLOSED=false
```

This allows hooks to warn but not block on missing signatures.

### Full Setup (Production Mode)

For CI/CD or production deployments:

1. **Install cosign**
   ```bash
   brew install sigstore/tap/cosign  # macOS
   ```

2. **Generate signing keys**
   ```bash
   cosign generate-key-pair
   mv cosign.pub .claude/
   # Store cosign.key securely (NOT in git)
   ```

3. **Configure environment**
   ```bash
   export QA_COSIGN_PUBLIC_KEY="$PWD/.claude/cosign.pub"
   ```

4. **Sign attestation**
   ```bash
   cosign sign-blob --key cosign.key \
     --output-signature .claude/verification/qa-attestation.sig \
     .claude/verification/qa-attestation.json
   ```

See `docs/guides/SUPPLY_CHAIN_SETUP.md` for detailed instructions.

## File Descriptions

### Attestation Files (Auto-generated by hooks)

| File | Purpose |
|------|---------|
| `qa-attestation.json` | SLSA provenance with quality metrics |
| `supply-chain-report.json` | Supply chain verification results |
| `verifier-dispute-gate.json` | Dispute gate check results |

| File | Purpose |
|------|---------|
| `qa-attestation.sig` | Cosign signature of attestation |
| `qa-attestation.bundle.json` | Rekor transparency log bundle |

### Governance Files

| File | Purpose |
|------|---------|
| `disputes.jsonl` | Append-only log of verifier disputes |

## Verification Workflow

1. **Quality gates run** → Generate `qa-attestation.json`
2. **Cosign signs** → Create `qa-attestation.sig`
3. **Rekor records** → Upload to transparency log → Create `qa-attestation.bundle.json`
4. **Hooks verify** → Check signature validity and Rekor entry
5. **Gates pass/fail** → Documented in verification reports

## Dispute Process

If you believe a quality gate decision is incorrect:

1. Add entry to `disputes.jsonl`:
   ```json
   {
     "dispute_id": "dispute-2026-02-14-001",
     "filed_at": "2026-02-14T10:00:00Z",
     "filed_by": "developer@example.com",
     "gate": "mypy-strict",
     "status": "open",
     "claim": "False positive in external type stub",
     "evidence": ["mypy-output.txt", "upstream-issue-12345"]
   }
   ```

2. Disputes are reviewed within 14 days (configurable)

3. Resolution updates the dispute record with status change

See `VERIFICATION_POLICY.md` for full workflow.
