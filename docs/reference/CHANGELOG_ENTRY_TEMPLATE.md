# Changelog Entry Template

Use this snippet inside `## [Unreleased]` in `CHANGELOG.md`.

```md
### Added
- <what was added and why it matters>

### Changed
- <what behavior changed>

### Deprecated
- <what is deprecated and replacement>

### Removed
- <what was removed>

### Fixed
- <what bug was fixed>

### Security
- <security fix or hardening>
```

Guidelines:
- Keep each bullet to one user-visible change.
- Use present or past tense consistently.
- Omit empty categories in actual release notes.
