# Feature Utilization Summary

## Quick Reference

### Supabase Features
- **PostgreSQL Database**: ✅ **HIGH** (Full CRUD, RPC, RLS).
- **pgvector**: ✅ **HIGH** (Similarity search, RAG).
- **Realtime**: ⚠️ **LOW** (Limited use).
- **Storage**: ⚠️ **LOW** (Limited use).
- **Auth**: ⚠️ **LOW** (Mostly WorkOS).

### Upstash Redis Features
- **Key-Value**: ✅ **HIGH** (General caching).
- **Caching**: ✅ **HIGH** (Embeddings, Tokens).
- **Rate Limiting**: ✅ **HIGH** (Distributed INCR + EXPIRE).
- **Streams/Queues**: ❌ **NONE** (Using Supabase table instead).

### Vercel Features
- **Serverless Fn**: ✅ **HIGH** (Python ASGI).
- **Vercel Cron**: ✅ **MEDIUM** (Queue processing).
- **Vercel Sandbox**: ✅ **HIGH** (Isolated execution).
- **Edge Functions**: ❌ **NONE** (Not used).

---

## Detailed Operations

### Database
- **Core Tables**: `organizations`, `projects`, `documents`, `requirements`, `tests`.
- **System Tables**: `mcp_sessions`, `embedding_queue`, `search_index`.
- **Advanced**: Custom RPC functions for complex entity searches and aggregations.

### Redis Caching
- **Embedding Cache**: Saves ~$0.02 per Vertex AI API hit; 24h TTL.
- **Token Cache**: Reduces JWKS validation latency; 1h TTL.
- **Response Cache**: Configurable caching for MCP tool outputs.

### Vercel Execution
- **Cron**: Runs every 5 minutes to process the `embedding_queue` table.
- **Sandbox**: ephemeral environments for executing Claude Agent SDK code and Python scripts safely.
