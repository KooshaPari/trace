# Feature Utilization Analysis: Supabase, Upstash, Vercel

## Executive Summary

The atomsAgent platform leverages a multi-cloud strategy using Supabase for persistence, Upstash for state/caching, and Vercel for execution. While core features (Database, Caching, Functions) are heavily utilized, several advanced capabilities like Realtime, Storage, and Edge Functions remain underutilized.

## 1. Supabase Utilization

| Feature | Status | Usage Level | Primary Use Cases |
|---------|--------|-------------|-------------------|
| **PostgreSQL** | ✅ Used | **HIGH** | Full CRUD, RPC functions, RLS. |
| **pgvector** | ✅ Used | **HIGH** | Semantic search, RAG, similarity. |
| **Realtime** | ✅ Implemented | **LOW** | Infrastructure exists; limited production use. |
| **Storage** | ✅ Implemented | **LOW** | Infrastructure exists; limited production use. |
| **Auth** | ⚠️ Partial | **LOW** | Mostly using WorkOS/AuthKit instead. |

## 2. Upstash Redis Utilization

| Feature | Status | Usage Level | Primary Use Cases |
|---------|--------|-------------|-------------------|
| **Key-Value** | ✅ Used | **HIGH** | Embedding cache, token cache. |
| **Rate Limiting** | ✅ Used | **HIGH** | Distributed sliding window limiting. |
| **Pub/Sub** | ❌ Not Used | **NONE** | - |
| **Streams** | ❌ Not Used | **NONE** | Using Supabase table for queues instead. |

## 3. Vercel Utilization

| Feature | Status | Usage Level | Primary Use Cases |
|---------|--------|-------------|-------------------|
| **Serverless Fn** | ✅ Used | **HIGH** | Python runtime, ASGI deployment. |
| **Vercel Cron** | ✅ Used | **MEDIUM** | Embedding queue processing (5 min). |
| **Vercel Sandbox** | ✅ Used | **HIGH** | Isolated code execution (agentapi). |
| **Edge Fn** | ❌ Not Used | **NONE** | - |

---

## Cost & Performance Drivers

- **High Cost**: Supabase (vector search CPU), Upstash (high request volume), Vercel Sandbox (allocation time).
- **Optimization Priority**: Migrate the embedding queue from Supabase to Redis Streams to reduce database load and improve task throughput.
