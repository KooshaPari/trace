## Item Commands

Manage items across all views (FEATURE, CODE, WIREFRAME, API, TEST, DATABASE, ROADMAP, PROGRESS).

### `rtm item create`

Create a new item in a specific view.

**Syntax:**
```bash
rtm item create <title> --view <VIEW> --type <TYPE> [OPTIONS]
```

**Options:**
- `--view, -v` (required): View (FEATURE, CODE, WIREFRAME, API, TEST, DATABASE, ROADMAP, PROGRESS)
- `--type, -t` (required): Item type (epic, feature, story, task, bug, etc.)
- `--description, -d`: Item description
- `--status, -s`: Status (default: todo)
- `--priority`: Priority (low, medium, high) (default: medium)
- `--owner, -o`: Item owner
- `--parent, -p`: Parent item ID
- `--metadata, -m`: Metadata as JSON
- `--local`: Force local-only operation (no sync)
- `--project`: Project directory path

**Examples:**
```bash
# Create a feature epic
rtm item create "User Authentication" --view FEATURE --type epic

# Create with description and owner
rtm item create "Login API" --view API --type endpoint \
  --description "REST endpoint for user login" --owner alice

# Create with metadata
rtm item create "Test Suite" --view TEST --type test_suite \
  --metadata '{"framework": "pytest", "coverage": 85}'

# Create in specific project
rtm item create "New Feature" --view FEATURE --type feature \
  --project /path/to/project
```

**Expected Output:**
```
вң“ Created EPIC-001: User Authentication
  File: .trace/epics/EPIC-001.md
  Project: my-project
  Indexed: /Users/user/.tracertm/tracertm.db
  Queued for sync to remote
```

### `rtm item show`

Show detailed information about an item.

**Syntax:**
```bash
rtm item show <ITEM_ID> [OPTIONS]
```

**Options:**
- `--metadata, -m`: Show metadata
- `--view, -v`: Show in specific view context
- `--depth, -d`: Drill-down depth (show children) (default: 1)
- `--version`: Show specific version
- `--children, -c`: Show all children
- `--tree, -t`: Show hierarchy tree (ASCII)
- `--ancestors, -a`: Show ancestor path

**Examples:**
```bash
# Basic item details
rtm item show EPIC-001

# Show with metadata
rtm item show EPIC-001 --metadata

# Show hierarchy tree
rtm item show EPIC-001 --tree

# Show ancestor path
rtm item show STORY-001 --ancestors
```

## Link Commands

Manage traceability links between items across views.

### `rtm link create`

Create a traceability link.

**Syntax:**
```bash
rtm link create <SOURCE_ID> <TARGET_ID> --type <TYPE> [OPTIONS]
```

**Link Types:**
- `implements` - Code implements requirement
- `tests` - Test verifies requirement
- `designs` - Design represents requirement
- `depends_on` - Depends on another item
- `blocks` - Blocks another item
- `related_to` - General relationship
- `decomposes_to` - Parent decomposes to child

**Options:**
- `--type, -t` (required): Link type
- `--metadata, -m`: Metadata as JSON

**Examples:**
```bash
# Create implementation link
rtm link create STORY-001 CODE-001 --type implements

# Create test link
rtm link create STORY-001 TEST-001 --type tests

# Create dependency
rtm link create TASK-001 TASK-002 --type depends_on

# With metadata
rtm link create STORY-001 DESIGN-001 --type designs \
  --metadata '{"figma_url": "https://..."}'
```

**Expected Output:**
```
вң“ Created link: User Login --[implements]--> LoginComponent
  Link ID: xyz789
```

### `rtm sync resolve`

Resolve a specific conflict.

**Syntax:**
```bash
rtm sync resolve <CONFLICT_ID> [OPTIONS]
```

**Strategies:**
- `last_write_wins`: Use newest timestamp (default)
- `local_wins`: Always prefer local version
- `remote_wins`: Always prefer remote version
- `manual`: Provide merged data with --data

**Options:**
- `--strategy, -s`: Resolution strategy (default: last_write_wins)
- `--data`: Merged data as JSON (for manual strategy)

**Examples:**
```bash
# Automatic resolution
rtm sync resolve conflict_abc123

# Use local version
rtm sync resolve conflict_abc123 --strategy local_wins
```

```bash
rtm item create "LoginComponent" --view CODE --type file

rtm link create STORY-001 CODE-001 --type implements

rtm progress show --item EPIC-001
```

```bash
# Check sync status
rtm sync status

# Work on items
rtm item list --status todo --owner me
rtm item update STORY-001 --status in_progress

rtm search "authentication" --view CODE
```

## Tips and Best Practices

1. **Use External IDs**: Items are automatically assigned external IDs (EPIC-001, STORY-042, etc.) for easy reference
2. **Leverage Search**: Use `rtm search` for quick lookups, `rtm query` for structured queries
3. **Check Impact**: Before major changes, use `rtm link impact` to see what will be affected
4. **Monitor Progress**: Regularly check `rtm progress report` to track velocity
5. **Sync Frequently**: Run `rtm sync` often to avoid conflicts
6. **Use Filters**: Most list commands support filters to narrow results
7. **JSON Output**: Add `--json` to most commands for programmatic use
8. **Shell Completion**: Install shell completion for faster command entry
