# Relationships API

Links (relationships) establish traceability between items, enabling impact analysis, coverage tracking, and compliance reporting. This API manages the creation, querying, and analysis of traceability relationships.

## Endpoints Overview

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | `/v1/projects/{project_id}/links` | List links |
| `POST` | `/v1/projects/{project_id}/links` | Create link |
| `GET` | `/v1/projects/{project_id}/links/{link_id}` | Get link |
| `DELETE` | `/v1/projects/{project_id}/links/{link_id}` | Delete link |
| `POST` | `/v1/projects/{project_id}/links/batch` | Batch create links |
| `GET` | `/v1/projects/{project_id}/items/{item_id}/links` | Get item's links |
| `GET` | `/v1/projects/{project_id}/trace` | Trace path between items |
| `GET` | `/v1/projects/{project_id}/impact-analysis` | Impact analysis |

## The Link Object

```json
{
  "id": "link_abc123xyz",
  "project_id": "proj_abc123",
  "source_id": "item_req_001",
  "target_id": "item_tc_001",
  "type": "verified_by",
  "direction": "forward",
  "status": "active",
  "description": "Test case verifies timing requirement",
  "metadata": {
    "confidence": "high",
    "verification_method": "test"
  },
  "created_by": "user_analyst123",
  "created_at": "2024-03-15T10:30:00Z",
  "updated_at": "2024-03-15T10:30:00Z"
}
```

### Link Fields

| Field | Type | Description |
|-------|------|-------------|
| `id` | string | Unique link identifier (prefix: `link_`) |
| `project_id` | string | Parent project ID |
| `source_id` | string | Source item ID |
| `target_id` | string | Target item ID |
| `type` | string | Relationship type |
| `direction` | string | `forward`, `backward`, `bidirectional` |
| `status` | string | `active`, `suspect`, `obsolete` |
| `description` | string | Optional description |
| `metadata` | object | Custom link data |
| `created_by` | string | Creator user ID |
| `created_at` | string | ISO 8601 creation timestamp |
| `updated_at` | string | ISO 8601 last update timestamp |

### Link Types

| Type | Description | Example |
|------|-------------|---------|
| `satisfies` | Requirement satisfies parent requirement | Child req â†’ Parent req |
| `derives_from` | Item derived from another | Design â†’ Requirement |
| `implements` | Implementation of specification | Code â†’ Design |
| `verified_by` | Verified by test | Requirement â†’ Test case |
| `validates` | Test validates requirement | Test case â†’ Requirement |
| `traces_to` | General traceability | Any â†’ Any |
| `depends_on` | Dependency relationship | Item â†’ Dependency |
| `relates_to` | Related items | Item â†’ Related item |
| `duplicates` | Duplicate of another | Item â†’ Original |
| `blocks` | Blocks another item | Blocker â†’ Blocked |

### Link Status

| Status | Description |
|--------|-------------|
| `active` | Link is current and valid |
| `suspect` | Source or target changed; needs review |
| `obsolete` | Link no longer valid |

### SDK Examples

#### Python

```python
from tracertm import TraceRTMClient

client = TraceRTMClient(api_key="rtm_live_sk_...")

# List verification links
links = client.links.list(
    project_id="proj_abc123",
    type="verified_by"
)

for link in links.data:
    print(f"{link.source.item_number} â†’ {link.target.item_number}")

# Find suspect links
suspect_links = client.links.list(
    project_id="proj_abc123",
    status="suspect"
)

print(f"Found {len(suspect_links.data)} suspect links to review")

# Links from a specific requirement type to test cases
links = client.links.list(
    project_id="proj_abc123",
    source_type="requirement",
    target_type="test_case"
)
```

#### TypeScript

```typescript
import { TraceRTMClient } from '@tracertm/sdk';

const client = new TraceRTMClient({ apiKey: 'rtm_live_sk_...' });

}
```

#### Go


## Create Link

Create a traceability link between items.

```http
POST /v1/projects/{project_id}/links
```

### Request Body

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `source_id` | string | Yes | Source item ID |
| `target_id` | string | Yes | Target item ID |
| `type` | string | Yes | Relationship type |
| `description` | string | No | Link description |
| `metadata` | object | No | Custom link data |

### Request

```bash
curl -X POST "https://api.tracertm.dev/v1/projects/proj_abc123/links" \
  -H "Authorization: Bearer rtm_live_sk_..." \
  -H "Content-Type: application/json" \
  -d '{
    "source_id": "item_req_002",
    "target_id": "item_tc_005",
    "type": "verified_by",
    "description": "Test case verifies error handling requirement",
    "metadata": {
      "confidence": "high",
      "verification_method": "test"
    }
  }'
```

### Response

```json
{
  "data": {
    "id": "link_new123xyz",
    "project_id": "proj_abc123",
    "source_id": "item_req_002",
    "target_id": "item_tc_005",
    "type": "verified_by",
    "direction": "forward",
    "status": "active",
    "description": "Test case verifies error handling requirement",
    "metadata": {
      "confidence": "high",
      "verification_method": "test"
    },
    "created_by": "user_analyst123",
    "created_at": "2024-12-08T10:00:00Z"
  }
}
```

### SDK Examples

#### Python

```python
link = client.links.create(
    project_id="proj_abc123",
    source_id="item_req_002",
    target_id="item_tc_005",
    type="verified_by",
    description="Test verifies error handling",
    metadata={"confidence": "high"}
)

print(f"Created link: {link.id}")
```

#### TypeScript

```typescript
const link = await client.links.create({
  projectId: 'proj_abc123',
  sourceId: 'item_req_002',
  targetId: 'item_tc_005',
  type: 'verified_by',
  description: 'Test verifies error handling'
});

console.log(`Created link: ${link.id}`);
```

## Batch Create Links

Create multiple links in a single request.

```http
POST /v1/projects/{project_id}/links/batch
```

### Request Body

```json
{
  "links": [
    {
      "source_id": "item_req_001",
      "target_id": "item_tc_001",
      "type": "verified_by"
    },
    {
      "source_id": "item_req_001",
      "target_id": "item_tc_002",
      "type": "verified_by"
    },
    {
      "source_id": "item_req_002",
      "target_id": "item_design_001",
      "type": "implements"
    }
  ]
}
```

### Request

```bash
curl -X POST "https://api.tracertm.dev/v1/projects/proj_abc123/links/batch" \
  -H "Authorization: Bearer rtm_live_sk_..." \
  -H "Content-Type: application/json" \
  -d '{
    "links": [
      {"source_id": "item_req_001", "target_id": "item_tc_001", "type": "verified_by"},
      {"source_id": "item_req_001", "target_id": "item_tc_002", "type": "verified_by"},
      {"source_id": "item_req_002", "target_id": "item_tc_003", "type": "verified_by"}
    ]
  }'
```

### Response

```json
{
  "data": {
    "results": [
      {"index": 0, "success": true, "id": "link_new001"},
      {"index": 1, "success": true, "id": "link_new002"},
      {"index": 2, "success": false, "error": {"code": "LINK_EXISTS", "message": "Link already exists"}}
    ],
    "summary": {
      "total": 3,
      "succeeded": 2,
      "failed": 1
    }
  }
}
```

### SDK Examples

#### Python

```python
results = client.links.batch_create(
    project_id="proj_abc123",
    links=[
        {"source_id": "item_req_001", "target_id": "item_tc_001", "type": "verified_by"},
        {"source_id": "item_req_001", "target_id": "item_tc_002", "type": "verified_by"},
        {"source_id": "item_req_002", "target_id": "item_tc_003", "type": "verified_by"}
    ]
)

print(f"Created {results.summary.succeeded}/{results.summary.total} links")
```

## Trace Path

Find traceability paths between two items.

```http
GET /v1/projects/{project_id}/trace
```

### Query Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `from` | string | Yes | Starting item ID |
| `to` | string | Yes | Ending item ID |
| `max_depth` | integer | No | Maximum path depth (default: 10) |
| `link_types` | string | No | Comma-separated allowed link types |

### Request

```bash
curl "https://api.tracertm.dev/v1/projects/proj_abc123/trace?from=item_req_001&to=item_tc_005&max_depth=5" \
  -H "Authorization: Bearer rtm_live_sk_..."
```

### Response

```json
{
  "data": {
    "from": "item_req_001",
    "to": "item_tc_005",
    "paths_found": 2,
    "paths": [
      {
        "length": 2,
        "path": [
          {"item_id": "item_req_001", "item_number": "VCS-REQ-001"},
          {"link_id": "link_001", "type": "verified_by"},
          {"item_id": "item_tc_001", "item_number": "VCS-TC-001"},
          {"link_id": "link_010", "type": "depends_on"},
          {"item_id": "item_tc_005", "item_number": "VCS-TC-005"}
        ]
      },
      {
        "length": 3,
        "path": [
          {"item_id": "item_req_001", "item_number": "VCS-REQ-001"},
          {"link_id": "link_002", "type": "implements"},
          {"item_id": "item_design_001", "item_number": "VCS-DES-001"},
          {"link_id": "link_011", "type": "verified_by"},
          {"item_id": "item_tc_003", "item_number": "VCS-TC-003"},
          {"link_id": "link_012", "type": "relates_to"},
          {"item_id": "item_tc_005", "item_number": "VCS-TC-005"}
        ]
      }
    ]
  }
}
```

### SDK Examples

#### Python

```python
trace = client.links.trace(
    project_id="proj_abc123",
    from_item="item_req_001",
    to_item="item_tc_005",
    max_depth=5
)

print(f"Found {trace.paths_found} paths")
for i, path in enumerate(trace.paths):
    print(f"Path {i+1} (length {path.length}):")
    for step in path.path:
        if "item_id" in step:
            print(f"  [{step['item_number']}]")
        else:
            print(f"  --{step['type']}-->")
```

## Link Validation

The API enforces these link constraints:

| Constraint | Description |
|------------|-------------|
| No self-links | Source and target must be different |
| No duplicates | Same source/target/type combination |
| Valid items | Both items must exist in project |
| Cycle detection | Optional, configurable per project |

## Error Responses

### Link Not Found

```json
{
  "error": {
    "code": "LINK_NOT_FOUND",
    "message": "Link 'link_invalid' not found",
    "status": 404
  }
}
```

### Duplicate Link

```json
{
  "error": {
    "code": "LINK_EXISTS",
    "message": "A link of type 'verified_by' already exists between these items",
    "status": 409,
    "details": {
      "existing_link_id": "link_existing123"
    }
  }
}
```

### Cycle Detected

```json
{
  "error": {
    "code": "LINK_CYCLE_DETECTED",
    "message": "Creating this link would create a cycle",
    "status": 422,
    "details": {
      "cycle_path": ["item_req_001", "item_design_001", "item_req_002", "item_req_001"]
    }
  }
}
```
