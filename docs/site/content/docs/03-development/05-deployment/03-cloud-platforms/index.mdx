# Cloud Platform Deployment

Deploy TraceRTM on major cloud platforms.

## AWS Deployment

### ECS Deployment

```yaml
# task-definition.json
{
  "family": "tracertm-api",
  "containerDefinitions": [{
    "name": "api",
    "image": "tracertm:2.4.0",
    "portMappings": [{
      "containerPort": 8000
    }],
    "environment": [
      {"name": "DATABASE_URL", "value": "..."}
    ]
  }]
}
```

### RDS Database

```bash
# Create RDS instance
aws rds create-db-instance \
  --db-instance-identifier tracertm-db \
  --db-instance-class db.t3.medium \
  --engine postgres \
  --master-username postgres \
  --master-user-password password \
  --allocated-storage 100
```

## Azure Deployment

### App Service

```bash
# Create App Service
az webapp create \
  --resource-group tracertm-rg \
  --plan tracertm-plan \
  --name tracertm-api

# Deploy
az webapp deployment source config-zip \
  --resource-group tracertm-rg \
  --name tracertm-api \
  --src app.zip
```

### Azure Database

```bash
# Create PostgreSQL server
az postgres server create \
  --resource-group tracertm-rg \
  --name tracertm-db \
  --admin-user postgres \
  --admin-password password \
  --sku-name GP_Gen5_2
```

## GCP Deployment

### Cloud Run

```bash
# Deploy to Cloud Run
gcloud run deploy tracertm-api \
  --image gcr.io/project/tracertm:2.4.0 \
  --platform managed \
  --region us-central1 \
  --allow-unauthenticated
```

### Cloud SQL

```bash
# Create Cloud SQL instance
gcloud sql instances create tracertm-db \
  --database-version=POSTGRES_14 \
  --tier=db-f1-micro \
  --region=us-central1
```

## Common Configuration

### Environment Variables

```bash
# Set in cloud platform
DATABASE_URL=postgresql://...
SECRET_KEY=your-secret-key
REDIS_URL=redis://...
```

### Health Checks

All platforms support health check endpoints:
- `/health` - Basic health
- `/health/db` - Database health
