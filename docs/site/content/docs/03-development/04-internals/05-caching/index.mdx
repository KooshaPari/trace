## Role in the architecture
- Use Redis for hot metadata and session-like data.
- Set TTLs and invalidation hooks on writes.
- Avoid cache stampedes with jitter/backoff.

## Considerations
- Throughput vs. latency trade-offs.
- Failure domains and retry/backoff strategies.
- Data model ownership and boundaries.
