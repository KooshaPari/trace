# Use Cases

Common use cases and scenarios for TraceRTM requirements traceability management.

## Software Development

### Agile Teams

TraceRTM helps agile teams maintain traceability across sprints and releases.

**Key Features:**
- Link user stories to test cases
- Track requirements through iterations
- Maintain traceability across sprints

[Learn more about Agile Teams →](/docs/wiki/use-cases/agile-teams)

### Waterfall Projects

Support traditional waterfall development with formal requirements traceability.

**Key Features:**
- Requirements → Design → Implementation → Testing
- Formal traceability matrices
- Compliance documentation

[Learn more about Waterfall Projects →](/docs/wiki/use-cases/waterfall-projects)

## Healthcare & Medical Devices

### Regulatory Compliance

Maintain traceability for FDA, ISO 13485, and other medical device regulations.

**Key Features:**
- Link requirements to test cases

- Audit trail maintenance

[Learn more about Healthcare →](/docs/wiki/use-cases/healthcare)

## Common Workflows

### Requirements to Test Cases

1. Create requirement items
2. Create test case items
3. Link requirements to test cases
4. Track test coverage
5. Generate traceability reports

### Impact Analysis

1. Identify changed requirement
2. Find all linked items
3. Analyze impact scope
4. Notify stakeholders
5. Update affected items

### Compliance Reporting

1. Organize requirements by standard
2. Link to verification evidence
3. Generate compliance matrix
4. Export for audits
