## Key Frameworks Reference

### Framework Summary

| Framework | Source | Key Metric | Use When |
|-----------|--------|-----------|----------|
| ISTQB | Certification board | Severity/Priority Matrix | Classifying defects |
| ODC | IBM Research | Removal Efficiency | Improving process |
| 5 Whys | Toyota | Root Cause | Simple analysis |
| Fishbone | Ishikawa | 6M Categories | Complex problems |
| FMEA | Engineering | RPN (Severity×Occurrence×Detection) | Critical systems |
| Pareto | Economics | 80/20 Rule | Resource allocation |
| CVSS | NIST | 0-10 Score | Security defects |
| CWE | MITRE | Weakness Type | Vulnerability category |
| SQALE | SonarSource | Debt Rating (A-F) | Technical debt |

## Common Workflows

### Workflow 1: Triage New Defect (5 minutes)

Use: **DEFECT_SPECIFICATION_QUICK_REFERENCE.md**

1. Read defect title and description
2. Use Section 1 (Severity-Priority Matrix) to assign severity/priority
3. Assign quality score using Section 6 checklist
4. Assign to owner using priority
5. Set SLA using Section 1

### Workflow 2: Root Cause Analysis (30 minutes)

Use: **DEFECT_SPECIFICATION_RESEARCH.md** Section 3 + **DEFECT_SPECIFICATION_EXAMPLES.md**

1. Choose RCA method (Five Whys, Fishbone, FMEA)
2. Review example in EXAMPLES.md
3. Document findings in RCA fields
4. Define preventive actions
5. Estimate effort to prevent

### Workflow 3: Performance Analysis (45 minutes)

Use: **DEFECT_SPECIFICATION_EXAMPLES.md** Example 2

1. Capture performance baseline (before/after)
2. Identify bottleneck
3. Calculate business impact
4. Map metrics (queries, response time, etc.)
5. Prioritize based on revenue impact

### Workflow 4: Security Vulnerability (20 minutes)

Use: **DEFECT_SPECIFICATION_QUICK_REFERENCE.md** Section 4 + **DEFECT_SPECIFICATION_EXAMPLES.md** Example 1

1. Identify CWE category
2. Calculate CVSS score
3. Determine SLA (24h for critical)
4. Flag for immediate remediation
5. Plan security testing

### Workflow 5: Identify Hotspots (60 minutes)

Use: **DEFECT_SPECIFICATION_QUICK_REFERENCE.md** Section 9 + **DEFECT_SPECIFICATION_EXAMPLES.md** Example 4

1. Collect all defects from last month
2. Count by module
3. Calculate cumulative percentage
4. Identify vital few (80/20)
5. Plan improvement actions
