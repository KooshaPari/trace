ARCHITECTURE
═══════════════════════════════════════════════════════════════════════════════

SEARCH FLOW:

  User Query
      ↓
  Full-Text Search (PostgreSQL FTS)
      ↓
  Vector Search (pgvector + embeddings)
      ↓
  Merge Results (RRF/weighted)
      ↓
  Rerank (VoyageAI rerank-2.5)
      ↓
  Return Top-K Results

BACKGROUND INDEXING:

  New Item Created
      ↓
  Item Added to Queue
      ↓
  Worker Pool (3 workers)
      ↓
  Batch Processing (50 items)
      ↓
  Generate Embeddings (VoyageAI)
      ↓
  Update Database (pgvector)
      ↓
  Track Cost & Progress

       RerankEnabled:     true,
   })

4. SEARCH:
   results, _ := searchEngine.Search(ctx, &search.SearchRequest{
       Query: "user authentication",
       Type:  search.SearchTypeHybrid,
       Limit: 20,
   })

DATABASE SETUP
═══════════════════════════════════════════════════════════════════════════════

-- Enable pgvector extension
CREATE EXTENSION IF NOT EXISTS vector;

-- Add embedding column (1024 dimensions for voyage-3.5)
ALTER TABLE items
ADD COLUMN embedding vector(1024);

-- Create index (choose one)
CREATE INDEX items_embedding_idx ON items
USING ivfflat (embedding vector_cosine_ops) WITH (lists = 100);

-- Or HNSW for better performance (PostgreSQL 16+)
CREATE INDEX items_embedding_hnsw_idx ON items
USING hnsw (embedding vector_cosine_ops);

TROUBLESHOOTING QUICK REFERENCE
═══════════════════════════════════════════════════════════════════════════════

ISSUE: No embeddings generated
CHECK:
  - INDEXER_ENABLED=true
  - Indexer running: indexer.IsRunning()
  - API key is valid
  - Database has pgvector extension
  - SELECT COUNT(*) FROM items WHERE embedding IS NULL;

ISSUE: Rate limit errors
SOLUTION:
  EMBEDDING_RATE_LIMIT=100
  INDEXER_WORKERS=1
  INDEXER_POLL_INTERVAL=60

ISSUE: High costs
SOLUTION:
  VOYAGE_MODEL=voyage-3.5
  RERANK_ENABLED=false
  Switch to OpenRouter: EMBEDDING_PROVIDER=openrouter

ISSUE: Slow searches
SOLUTION:
  RERANK_ENABLED=false
  Check indices: SELECT * FROM pg_indexes WHERE tablename='items';
  Use HNSW index instead of IVFFlat

ISSUE: pgvector not found
SOLUTION:
  psql: CREATE EXTENSION vector;
  If fails, install pgvector on system first

MONITORING COMMANDS
═══════════════════════════════════════════════════════════════════════════════

CHECK EMBEDDING STATUS:
  SELECT COUNT(*) as total,
         COUNT(embedding) as embedded,
         COUNT(*) - COUNT(embedding) as pending
  FROM items;

FIND UNEMBEDDED ITEMS:
  SELECT id, title
  FROM items
  WHERE embedding IS NULL
  LIMIT 10;

CHECK INDEX USAGE:
  SELECT indexname, idx_scan
  FROM pg_stat_user_indexes
  WHERE tablename = 'items'
  AND indexname LIKE '%embedding%';
