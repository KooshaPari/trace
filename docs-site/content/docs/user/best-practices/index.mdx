# Best Practices

Best practices for using TraceRTM effectively in your projects.

## Project Organization

### 1. Use Clear Naming Conventions

```python
# Good - descriptive names
"User Authentication Requirement"
"Login Test Case - Valid Credentials"
"Password Reset Design Document"

# Bad - unclear names
"Req 1"
"Test"
"Doc"
```

### 2. Organize by Views

Use views to organize different artifact types:

- **requirements**: System requirements
- **test-cases**: Test specifications
- **design**: Design documents
- **code**: Code references

### 3. Use Status Workflows

Implement consistent status workflows:

```
requirements: draft → review → approved → implemented
test-cases: draft → ready → executed → passed
```

## Link Management

### 1. Create Meaningful Links

```python
# Good - clear relationship
requirement --[verified_by]--> test_case
design --[implements]--> requirement
requirement --[depends_on]--> requirement

# Avoid - unclear relationships
item --[related_to]--> item  # Too vague
```

### 2. Maintain Bidirectional Awareness

While links are directional, document reverse relationships:

```python
# Forward link
requirement --[verified_by]--> test_case

# Document reverse in description
# "This test case verifies requirement REQ-001"
```

### 3. Avoid Circular Dependencies

Use cycle detection to prevent circular links:

```python
# Bad - creates cycle
A --[depends_on]--> B
B --[depends_on]--> C
C --[depends_on]--> A

# Good - acyclic
A --[depends_on]--> B
B --[depends_on]--> C
```

## Item Management

### 1. Write Clear Descriptions

```markdown
# Good - comprehensive description
## User Authentication Requirement

The system must authenticate users using secure credentials.

**Acceptance Criteria:**
- Users can log in with email/password
- Failed attempts are logged
- Account lockout after 5 failed attempts

# Bad - minimal description
User login
```

### 2. Use Metadata Effectively

```python
# Add useful metadata
metadata = {
    "priority": "high",
    "owner": "team-alpha",
    "sprint": "Sprint 5",
    "epic": "Authentication"
}
```

### 3. Keep Items Up to Date

- Update status as work progresses
- Add notes for important changes
- Maintain version history

## Search and Filtering

### 1. Use Consistent Keywords

```python
# Tag items with consistent keywords
tags = ["authentication", "security", "user-management"]
```

### 2. Leverage Full-Text Search

```python
# Search across titles and descriptions
search_query = "user authentication login"
```

### 3. Combine Filters

```python
# Filter by multiple criteria
items = client.items.list(
    project_id=project_id,
    view="requirements",
    status="approved",
    search="authentication"
)
```

## Performance

### 1. Paginate Large Results

```python
# Always paginate
items = client.items.list(
    project_id=project_id,
    skip=0,
    limit=50
)
```

### 2. Cache Frequently Accessed Data

```python
# Cache project metadata
project = cache.get(f"project:{project_id}")
if not project:
    project = client.projects.get(project_id)
    cache.set(f"project:{project_id}", project, ttl=3600)
```

### 3. Batch Operations

```python
# Batch create items
items = [Item(...) for _ in range(100)]
client.items.create_batch(items)
```

## Security

### 1. Use API Keys Securely

```python
# Store in environment variables
api_key = os.getenv("TRACERTM_API_KEY")

# Never commit to code
# ❌ api_key = "rtm_live_abc123..."
```

### 2. Implement Proper Authorization

```python
# Check permissions before operations
if not user.has_permission("items:write"):
    raise PermissionError("Insufficient permissions")
```

### 3. Audit Sensitive Operations

```python
# Log important operations
logger.info("Item deleted", extra={
    "item_id": item_id,
    "user_id": user_id,
    "timestamp": datetime.utcnow()
})
```
