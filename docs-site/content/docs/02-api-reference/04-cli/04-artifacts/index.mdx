## What this covers
- How to manage artifacts with the Trace CLI
- Expected inputs, flags, and outputs
- Safety checks before executing

## Quick commands
```bash
trace artifacts list --project <id>
```

```bash
trace artifacts create --project <id> --type code --name "Checkout Service"
```

```bash
trace artifacts link --from <code-id> --to <feature-id> --type implements
```

## Best practices
- Use `--project` to scope actions and avoid cross-project edits.
- Prefer idempotent operations (`trace ... upsert`) in automation.
- Pipe to jq for verification before destructive changes.
