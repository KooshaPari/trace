# Authentication Best Practices

Follow these security best practices to ensure secure authentication with the TraceRTM API.

## Credential Storage

### ✅ DO: Use Environment Variables

```bash
# Set in environment
export TRACERTM_API_KEY="rtm_live_abc123..."

# Use in code
import os
api_key = os.getenv("TRACERTM_API_KEY")
```

### ❌ DON'T: Hardcode Credentials

```python
# ❌ Bad: Hardcoded
api_key = "rtm_live_abc123..."

# ❌ Bad: In config files committed to git
# config.json
{
  "api_key": "rtm_live_abc123..."
}
```

### ✅ DO: Use Secret Management

```python
# ✅ Good: AWS Secrets Manager
import boto3
secrets = boto3.client('secretsmanager')
api_key = secrets.get_secret_value(
    SecretId='tracertm/api-key'
)['SecretString']

# ✅ Good: HashiCorp Vault
import hvac
client = hvac.Client(url='https://vault.example.com')
api_key = client.secrets.kv.v2.read_secret_version(
    path='tracertm'
)['data']['data']['api_key']
```

## API Key Management

### 1. Use Separate Keys

Create separate keys for different environments:

```bash
# Development
export TRACERTM_API_KEY_DEV="rtm_test_..."

# Staging
export TRACERTM_API_KEY_STAGING="rtm_test_..."

# Production
export TRACERTM_API_KEY_PROD="rtm_live_..."
```

### 2. Rotate Regularly

Rotate API keys periodically:

```bash
# Create new key
trace api-key create --name "New Production Key"

# Update applications

# Revoke old key after migration
trace api-key revoke <old-key-id>
```

### 3. Use Least Privilege

Grant only necessary permissions:

```bash
# ✅ Good: Specific scopes
trace api-key create \
  --scopes "items:read items:write"

# ❌ Bad: Overly broad
trace api-key create \
  --scopes "admin"
```

### 4. Monitor Key Usage

Review key usage regularly:

```bash
# Check key activity
trace api-key logs <key-id>

# Review permissions
trace api-key show <key-id>
```

## Token Security

### 1. Never Log Tokens

```python
# ❌ Bad: Logging tokens
logger.info(f"Using token: {access_token}")

# ✅ Good: Log token metadata
logger.info(f"Token expires at: {expires_at}")
```

### 2. Use HTTPS Only

Always use HTTPS in production:

```python
# ✅ Good: HTTPS
base_url = "https://api.tracertm.dev"

# ❌ Bad: HTTP
base_url = "http://api.tracertm.dev"  # Never!
```

### 3. Validate Token Expiration

Check token expiration before use:

```python
import jwt
from datetime import datetime

def is_token_valid(token):
    try:
        decoded = jwt.decode(
            token,
            options={"verify_signature": False}
        )
        exp = decoded.get('exp', 0)
        return datetime.now().timestamp() < exp
    except:
        return False
```

### 4. Implement Token Refresh

Automatically refresh expired tokens:

```python
def get_valid_token(access_token, refresh_token):
    if is_token_expired(access_token):
        return refresh_access_token(refresh_token)
    return access_token
```

## OAuth 2.0 Security

### 1. Protect Client Secret

Never expose client secret in client-side code:

```javascript
// ❌ Bad: Client secret in frontend
const clientSecret = "secret_xyz789...";

// ✅ Good: Server-side only
// Keep secret on backend
```

### 2. Use State Parameter

Always use state parameter for CSRF protection:

```python
import secrets

# Generate random state
state = secrets.token_urlsafe(32)

# Store in session
session['oauth_state'] = state

# Verify on callback
if request.args['state'] != session['oauth_state']:
    raise ValueError("Invalid state parameter")
```

### 3. Validate Redirect URI

Always validate redirect URI:

```python
REGISTERED_REDIRECT_URI = "https://yourapp.com/callback"

def handle_oauth_callback(redirect_uri, code):
    if redirect_uri != REGISTERED_REDIRECT_URI:
        raise ValueError("Invalid redirect URI")
    # Process callback
```

### 4. Handle Token Securely

Store tokens securely:

```python
# ✅ Good: Encrypted storage
from cryptography.fernet import Fernet

key = Fernet.generate_key()
cipher = Fernet(key)
encrypted_token = cipher.encrypt(token.encode())

# ❌ Bad: Plain text storage
tokens = {
    "access_token": token  # Unencrypted!
}
```

## Request Security

### 1. Validate Input

Always validate input before sending:

```python
def create_item(title, description):
    # Validate input
    if not title or len(title) > 255:
        raise ValueError("Invalid title")
    
    # Make request
    return api_client.create_item(title, description)
```

### 2. Use Idempotency Keys

For critical operations, use idempotency keys:

```python
import uuid

idempotency_key = str(uuid.uuid4())
response = requests.post(
    "https://api.tracertm.dev/v1/items",
    headers={
        "Authorization": f"Bearer {token}",
        "Idempotency-Key": idempotency_key
    },
    json={"title": "New Item"}
)
```

### 3. Handle Errors Gracefully

Implement proper error handling:

```python
try:
    response = api_client.create_item(title, description)
except RateLimitError:
    # Retry with backoff
    retry_with_backoff(api_client.create_item, title, description)
except AuthenticationError:
    # Refresh token and retry
    refresh_token()
    response = api_client.create_item(title, description)
except APIError as e:
    # Log and handle
    logger.error(f"API error: {e}")
    raise
```

## Network Security

### 1. Verify SSL Certificates

Always verify SSL certificates:

```python
import requests

# ✅ Good: Verify certificates (default)
response = requests.get(url, verify=True)

# ❌ Bad: Disable verification
response = requests.get(url, verify=False)  # Never!
```

### 2. Use Certificate Pinning

For high-security applications:

```python
import requests

# Pin certificate
response = requests.get(
    url,
    verify='/path/to/certificate.pem'
)
```

### 3. Implement Timeouts

Set appropriate timeouts:

```python
import requests

# Set timeout
response = requests.get(
    url,
    timeout=(5, 30)  # (connect, read)
)
```

## Monitoring and Auditing

### 1. Log Authentication Events

Log authentication attempts:

```python
import logging

logger = logging.getLogger(__name__)

def authenticate(api_key):
    try:
        result = verify_api_key(api_key)
        logger.info(f"Authentication successful: {api_key[:10]}...")
        return result
    except Exception as e:
        logger.warning(f"Authentication failed: {e}")
        raise
```

### 2. Monitor Unusual Activity

Monitor for suspicious patterns:

```python
# Track failed attempts
failed_attempts = {}

def check_suspicious_activity(api_key):
    key_prefix = api_key[:10]
    if key_prefix not in failed_attempts:
        failed_attempts[key_prefix] = 0
    
    failed_attempts[key_prefix] += 1
    
    if failed_attempts[key_prefix] > 5:
        # Alert security team
        send_security_alert(key_prefix)
```

### 3. Review Access Logs

Regularly review access logs:

```bash
# Review API key usage
trace api-key logs <key-id> --last 30d

# Check for anomalies
trace audit-logs --type authentication --last 7d
```

## Compliance

### 1. Follow Data Protection Regulations

- Encrypt sensitive data at rest
- Use secure transmission (HTTPS)
- Implement access controls
- Maintain audit logs

### 2. Regular Security Reviews

- Review API key permissions quarterly
- Audit authentication logs monthly
- Update security practices as needed

## Incident Response

### 1. Revoke Compromised Credentials

If credentials are compromised:

```bash
# Immediately revoke
trace api-key revoke <compromised-key-id>

# Create new key
trace api-key create --name "Replacement Key"
```

### 2. Rotate All Related Credentials

Rotate all credentials that may be affected:

```bash
# Rotate API keys
# Rotate OAuth client secrets
# Invalidate all tokens
```

### 3. Review Access Logs

Review logs for unauthorized access:

```bash
# Check for suspicious activity
trace audit-logs --last 24h --filter suspicious
```

## Summary

### Key Principles

1. **Never commit credentials** to version control
2. **Use environment variables** or secret management
3. **Rotate credentials** regularly
4. **Follow least privilege** principle
5. **Monitor and audit** authentication activity
6. **Use HTTPS** for all API communication
7. **Implement proper error handling**
8. **Validate all input** before sending

### Quick Checklist

- [ ] Credentials stored securely (not in code)
- [ ] HTTPS used for all requests
- [ ] API keys rotated regularly
- [ ] Least privilege permissions granted
- [ ] Token expiration handled
- [ ] Error handling implemented
- [ ] Monitoring and logging enabled
- [ ] Security reviews conducted
