# Concepts

TraceRTM is a governed graph of artifacts and relationships, exposed through role-specific views and powered by workflows and automation.

## Overview

- **Artifacts**: Feature, code, test, API, infra, wireframe, and more—with metadata, history, and ownership.
- **Relationships**: Typed, directional links (hierarchy, implementation, testing, dependency, operational) that power traceability and impact analysis.
- **Workflows**: State machines with guarded transitions, automation hooks, and audit trails.
- **Views**: 16 professional lenses over the same graph so teams see what matters without duplicating data.
- **Governance**: Validation rules, required links, and event-sourced audit for trustworthy data.

## How to use this model

- Model work as artifacts; avoid “misc” buckets.
- Link artifacts with the correct relationship type and direction.
- Let views filter/slice the graph instead of copying data.
- Automate transitions and checks; keep humans on decisions, not bookkeeping.
