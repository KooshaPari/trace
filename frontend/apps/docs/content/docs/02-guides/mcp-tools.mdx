# TraceRTM MCP/CLI/API Capability Matrix (Offline + Online)

## Scope

This doc maps **MCP**, **CLI**, and **API** capabilities in both **offline** and **online** modes, and clarifies that each client is optional.

## Key Principles

- **Offline-first:** Local DB is primary for local operations.
- **Online optional:** Sync/API only required for remote collaboration.
- **Client optionality:** MCP and CLI can be used independently (or together).
- **Auth separation:** MCP auth does **not** depend on CLI state.

## Clients at a Glance

### MCP (FastMCP server)

- **Entry:** `python -m tracertm.mcp` or `rtm mcp start`
- **Auth:** WorkOS AuthKit JWT (OAuth), optional static dev API keys
- **Offline:** Full CRUD + analysis via local DB
- **Online:** Sync via API if configured

### CLI (Typer + Rich)

- **Entry:** `rtm <command>`
- **Auth:** WorkOS AuthKit device flow
- **Offline:** Full CRUD + analysis via local DB
- **Online:** Sync/pull/push via API

### API

- **Local API client:** `src/tracertm/api/client.py` (DB-backed)
- **Remote sync client:** `src/tracertm/api/sync_client.py` (HTTP)
- **Auth:** Bearer token (AuthKit OAuth)

## Auth Matrix

| Client | Offline | Online | Auth Method                            |
| ------ | ------- | ------ | -------------------------------------- |
| MCP    | ✅      | ✅     | AuthKit JWT (OAuth) or static dev keys |
| CLI    | ✅      | ✅     | AuthKit device flow                    |
| API    | n/a     | ✅     | Bearer token (AuthKit OAuth)           |

## Storage & State

### Per-user system dir

- `~/.tracertm/`
  - `config.yaml` (global config)
  - `tracertm.db` (local DB)
  - `mcp/token.json` (optional MCP token storage)

### Per-project dir (repo)

- `.tracertm/`
  - `project.yaml` (project metadata)
  - `specs/` (optional exports)
  - `.gitignore` (ignore local state/cache)

## Offline + Online Flow

### Offline-only (MCP or CLI)

1. Local DB initialized in `~/.tracertm/`
2. Create projects/items/links/specs
3. Export/backup as needed

### Online Sync (optional)

1. Authenticate (CLI device flow or MCP OAuth)
2. Configure `api_url` and token
3. Run sync (push/pull changes)

## Notes

- MCP does **not** require CLI tokens; CLI token sync is optional and manual.
- Dev API keys are supported for MCP in non-production environments.
- Remote MCP deploy uses the same AuthKit OAuth bearer tokens as the API.
