# Testing Guide

TraceRTM uses a multi-layered testing approach to ensure code quality and reliability.

## Testing Pyramid

```
       /\
      /E2E\      End-to-End Tests (Playwright)
     /______\
    /        \
   /Integration\ Integration Tests (pytest + httpx)
  /____________\
 /              \
/   Unit Tests   \  Unit Tests (pytest + vitest)
/________________\
```

## Backend Testing

### Unit Tests

Test individual functions and classes in isolation.

**Location**: `tests/unit/`

**Tools**: pytest, pytest-asyncio, pytest-mock

```python
# tests/unit/services/test_item_service.py
import pytest
from tracertm.services import ItemService
from tracertm.schemas import ItemCreate

@pytest.fixture
def mock_repository(mocker):
    return mocker.Mock()

@pytest.fixture
def service(mock_repository):
    return ItemService(mock_repository)

async def test_create_item(service, mock_repository):
    # Arrange
    data = ItemCreate(title="Test Item")
    mock_repository.create.return_value = Item(
        id="1",
        title="Test Item",
        created_at=datetime.now()
    )

    # Act
    result = await service.create(data)

    # Assert
    assert result.title == "Test Item"
    mock_repository.create.assert_called_once()
```

### Integration Tests

Test API endpoints with real database.

**Location**: `tests/integration/`

**Tools**: pytest, httpx, TestClient

```python
# tests/integration/test_items_api.py
import pytest
from httpx import AsyncClient
from tracertm.api.main import app

@pytest.fixture
async def client():
    async with AsyncClient(app=app, base_url="http://test") as client:
        yield client

async def test_create_item(client):
    # Create item
    response = await client.post("/items/", json={
        "title": "Test Item",
        "description": "Test Description"
    })

    assert response.status_code == 200
    data = response.json()
    assert data["title"] == "Test Item"

    # Verify item exists
    item_id = data["id"]
    response = await client.get(f"/items/{item_id}")
    assert response.status_code == 200
```

## Frontend Testing

### Unit Tests

Test individual components and hooks.

**Location**: `src/__tests__/`

**Tools**: Vitest, Testing Library

```tsx
// src/__tests__/components/ItemCard.test.tsx
import { render, screen } from '@testing-library/react';
import { expect, test } from 'vitest';
import { ItemCard } from '@/components/ItemCard';

test('renders item title', () => {
  const item = {
    id: '1',
    title: 'Test Item',
    description: 'Test Description',
  };

  render(<ItemCard item={item} />);

  expect(screen.getByText('Test Item')).toBeInTheDocument();
  expect(screen.getByText('Test Description')).toBeInTheDocument();
});

test('calls onDelete when delete button clicked', async () => {
  const onDelete = vi.fn();
  const item = { id: '1', title: 'Test Item' };

  render(<ItemCard item={item} onDelete={onDelete} />);

  const deleteButton = screen.getByRole('button', { name: /delete/i });
  await userEvent.click(deleteButton);

  expect(onDelete).toHaveBeenCalledWith('1');
});
```

### Hook Testing

```tsx
// src/__tests__/hooks/useItems.test.ts
import { renderHook, waitFor } from '@testing-library/react';
import { QueryClient, QueryClientProvider } from '@tanstack/react-query';
import { useItems } from '@/hooks/useItems';

function createWrapper() {
  const queryClient = new QueryClient({
    defaultOptions: {
      queries: { retry: false },
    },
  });

  return ({ children }) => (
    <QueryClientProvider client={queryClient}>{children}</QueryClientProvider>
  );
}

test('fetches items', async () => {
  const { result } = renderHook(() => useItems(), {
    wrapper: createWrapper(),
  });

  await waitFor(() => expect(result.current.isSuccess).toBe(true));
  expect(result.current.data).toHaveLength(5);
});
```

## End-to-End Tests

Test complete user flows in a real browser.

**Location**: `frontend/apps/web/e2e/`

**Tools**: Playwright

```typescript
// e2e/items.spec.ts
import { test, expect } from '@playwright/test';

test.describe('Items Management', () => {
  test('creates, views, and deletes item', async ({ page }) => {
    // Navigate to items page
    await page.goto('/items');

    // Create new item
    await page.click('button:has-text("New Item")');
    await page.fill('input[name="title"]', 'Test Item');
    await page.fill('textarea[name="description"]', 'Test Description');
    await page.click('button[type="submit"]');

    // Verify item appears in list
    await expect(page.locator('text=Test Item')).toBeVisible();

    // View item details
    await page.click('text=Test Item');
    await expect(page.locator('text=Test Description')).toBeVisible();

    // Delete item
    await page.click('button:has-text("Delete")');
    await page.click('button:has-text("Confirm")');
    await expect(page.locator('text=Test Item')).not.toBeVisible();
  });

  test('filters items by type', async ({ page }) => {
    await page.goto('/items');
    await page.selectOption('select[name="type"]', 'requirement');
    await expect(page.locator('[data-type="requirement"]')).toBeVisible();
    await expect(page.locator('[data-type="test"]')).not.toBeVisible();
  });
});
```

## Mocking

### Backend Mocking

```python
# Using pytest-mock
def test_with_mock(mocker):
    mock_api = mocker.patch('tracertm.api.external_api')
    mock_api.fetch.return_value = {"data": "test"}

    result = service.fetch_data()
    assert result == {"data": "test"}
```

### Frontend Mocking

```tsx
// Using vitest
import { vi } from 'vitest';

// Mock API
vi.mock('@/api/client', () => ({
  api: {
    items: {
      list: vi.fn(() => Promise.resolve(mockItems)),
      create: vi.fn((data) => Promise.resolve({ id: '1', ...data })),
    },
  },
}));

test('uses mocked API', async () => {
  const { result } = renderHook(() => useItems());
  await waitFor(() => expect(result.current.isSuccess).toBe(true));
  expect(result.current.data).toEqual(mockItems);
});
```

## CI/CD Integration

### GitHub Actions

```yaml
# .github/workflows/test.yml
name: Tests

on: [push, pull_request]

jobs:
  backend-tests:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - uses: actions/setup-python@v4
        with:
          python-version: '3.12'
      - run: pip install uv
      - run: uv sync
      - run: pytest --cov --cov-report=xml
      - uses: codecov/codecov-action@v3

  frontend-tests:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - uses: oven-sh/setup-bun@v1
      - run: bun install
      - run: bun test --coverage
      - uses: codecov/codecov-action@v3

  e2e-tests:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - uses: oven-sh/setup-bun@v1
      - run: bun install
      - run: bunx playwright install --with-deps
      - run: bun run test:e2e
```

## Best Practices

1. **Write tests first** (TDD) - Think through requirements
2. **Test behavior, not implementation** - Focus on what, not how
3. **Keep tests independent** - No shared state between tests
4. **Use descriptive names** - Test name should describe what it tests
5. **Arrange-Act-Assert** - Clear test structure
6. **Mock external dependencies** - Keep tests fast and reliable
7. **Test edge cases** - Not just happy paths
8. **Maintain high coverage** - Aim for 80%+
9. **Run tests in CI** - Automate testing
10. **Keep tests fast** - Fast feedback loop
