# Team Onboarding Guide

Welcome to the Trace RTM project! This guide will help you get up to speed quickly.

## What is Trace RTM?

Trace is a comprehensive multi-view project management system optimized for power users orchestrating 1000+ AI agents. It provides:

- **16 views** covering all PM aspects (Feature, Code, Test, API, etc.)
- **Git-style versioning** with time-travel capability
- **NATS-based event coordination** for distributed task execution
- **Natural language assertions** and verification systems
- **Comprehensive traceability** with 60+ link types

## Key Architectural Decisions

### Why NATS Over Redis?

| Feature          | NATS JetStream                 | Redis                 |
| ---------------- | ------------------------------ | --------------------- |
| Message Delivery | At-least-once, exactly-once ✅ | At-most-once ❌       |
| Persistence      | Built-in JetStream ✅          | Requires AOF/RDB ⚠️   |
| Work Queues      | Pull consumers ✅              | Lua scripts ⚠️        |
| Clustering       | Native ✅                      | Complex sharding ⚠️   |
| DAG Coordination | Perfect fit ✅                 | Workarounds needed ❌ |

**Decision**: Use NATS for distributed event coordination and agent orchestration.

### Versioning Strategy

**Hybrid event sourcing + snapshots**:

- Events: Complete audit trail, time-travel capability
- Snapshots: Performance optimization (every 100 events)
- Git-style: Content-addressable commits with DAG structure
- Semantic: Automatic version bumping from conventional commits

### Storage Architecture

**SQLite → PostgreSQL migration path**:

- Start: SQLite for simplicity and local development
- Scale: PostgreSQL for production and multi-user scenarios
- Optional: Neo4j for graph queries on links
- Audit: Immutable NATS JetStream for event log

## Development Workflow

### 1. Create a Feature Branch

```bash
git checkout -b feature/your-feature-name
```

### 2. Make Changes

- Follow TDD (Test-Driven Development)
- Write tests first
- Implement feature
- Ensure all tests pass

### 3. Run Tests

```bash
# Backend tests
pytest

# Frontend tests
cd frontend/apps/web
bun test

# E2E tests
bun run test:e2e
```

### 4. Code Quality Checks

```bash
# Backend linting
ruff check .
ruff format .

# Frontend linting
bun run lint

# Type checking
ty check src/ --error-on-warning
```

### 5. Commit and Push

```bash
git add .
git commit -m "feat: your feature description"
git push origin feature/your-feature-name
```

### 6. Create Pull Request

- Use GitHub UI to create PR
- Ensure CI checks pass
- Request review from team

## Common Tasks

### Adding a New View

1. Define schema in `frontend/packages/types`
2. Create API endpoint in `src/tracertm/api/routers`
3. Add repository method in `src/tracertm/repositories`
4. Create React component in `frontend/apps/web/src/views`
5. Add route in TanStack Router
6. Write tests

### Adding a New MCP Tool

1. Define tool in `src/tracertm/mcp/tools`
2. Register in `src/tracertm/mcp/server.py`
3. Add schema validation
4. Write unit tests
5. Add integration tests
6. Update documentation

### Working with NATS

```python
from tracertm.infrastructure.nats import NATSClient

async with NATSClient() as client:
    # Publish event
    await client.publish("events.item.created", {
        "item_id": "123",
        "type": "requirement"
    })

    # Subscribe to events
    async for msg in client.subscribe("events.item.*"):
        print(f"Received: {msg.data}")
```
