# API Reference

Welcome to the TraceRTM API reference documentation. This section provides comprehensive documentation for all available API endpoints, auto-generated from the backend OpenAPI specification.

## Overview

The TraceRTM API is a RESTful API built with FastAPI that provides programmatic access to all traceability management features. All endpoints return JSON responses and follow standard HTTP status code conventions.

### Base URL

```
http://localhost:8000
```

### Authentication

Most endpoints require authentication. TraceRTM uses device-based OAuth 2.0 flow:

1. Request a device code from `/api/v1/auth/device/code`
2. User navigates to verification URL and enters the code
3. Poll `/api/v1/auth/device/token` to receive access token
4. Include token in `Authorization` header: `Bearer <token>`

See [Authentication](/api-reference/auth) for detailed endpoint documentation.

## Rate Limiting

The API implements rate limiting to ensure fair usage:

- Default: 100 requests per minute per user
- Burst: 200 requests per minute

Rate limit headers are included in responses:

```
X-RateLimit-Limit: 100
X-RateLimit-Remaining: 95
X-RateLimit-Reset: 1640000000
```

## Error Handling

The API uses standard HTTP status codes:

| Code | Description                             |
| ---- | --------------------------------------- |
| 200  | Success                                 |
| 201  | Created                                 |
| 400  | Bad Request - Invalid parameters        |
| 401  | Unauthorized - Missing or invalid token |
| 403  | Forbidden - Insufficient permissions    |
| 404  | Not Found                               |
| 422  | Validation Error - Invalid request body |
| 429  | Too Many Requests - Rate limit exceeded |
| 500  | Internal Server Error                   |

Error responses include details:

```json
{
  "detail": "Error message",
  "validation_errors": [...]  // For 422 errors
}
```
