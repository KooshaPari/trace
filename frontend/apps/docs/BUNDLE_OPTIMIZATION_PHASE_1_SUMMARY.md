## ⚠️ Blocked Tasks

### 4. Build Infrastructure Issues

**Current Blockers**:

1. **Module Resolution Errors**:
   - Cannot resolve '@/source' (not generated until build time)
   - Cannot resolve '@/components/mdx-components'
   - Cannot resolve '@/components/icon-sprite'

2. **Build Dependency Cycle**:
   - `prebuild` script runs `generate-openapi.ts` + `build-search-index.ts`
   - `build-search-index.ts` imports from `source.ts`
   - `source.ts` imports from `@/.source` (generated by Next.js build)
   - Circular dependency: build needs .source, but .source needs build

3. **Configuration Conflicts**:
   - Config consolidated to next.config.ts only
   - `swcMinify` deprecated in Next.js 15 (now default)

**Required Fixes**:

1. Fix circular build dependency by:
   - Moving search index build to post-build step, OR
   - Making search index build optional/conditional, OR
   - Generating .source independently before build

2. Consolidate Next.js config files:
   - Keep only next.config.ts
   - Remove swcMinify option (deprecated)
   - Verify @next/bundle-analyzer integration

3. Fix module resolution:
   - Ensure tsconfig paths are correct
   - Verify all required files exist
   - Check for missing components
