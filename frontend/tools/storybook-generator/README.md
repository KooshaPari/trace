# TraceRTM Storybook Story Generator

Automatically generates CSF 3.0 Storybook stories for React components with design system integration and requirements traceability.

## Features

- **Automatic Story Generation**: Scans component directories and generates type-safe stories
- **CVA Integration**: Detects class-variance-authority variants and generates stories for each
- **Props Extraction**: Uses TypeScript AST to extract component props and generate controls
- **Design Integration**: Links to Figma designs via metadata YAML
- **Requirements Tracing**: Adds trace IDs for linking stories back to requirements
- **CSF 3.0**: Modern Storybook format with autodocs support
- **Smart Defaults**: Generates sensible default values and showcase stories

## Installation

The generator is already set up in this workspace. Just install dependencies:

```bash
bun install
```

## Usage

### Generate All Stories

```bash
bun run generate:stories
```

### Generate for Specific Component

```bash
bun run generate:stories --component Button
```

### Force Regenerate (Overwrite Existing)

```bash
bun run generate:stories --force
```

### Verbose Output

```bash
bun run generate:stories --verbose
```

## Design Metadata

Create `.trace/.meta/designs.yaml` to link components to Figma designs:

```yaml
components:
  Button:
    figmaUrl: 'https://www.figma.com/file/xxx/Design-System?node-id=123'
    componentId: 'comp-button-001'
    storyId: 'story-button-001'
    variants:
      - default
      - destructive
      - outline

  Input:
    figmaUrl: 'https://www.figma.com/file/xxx/Design-System?node-id=456'
    componentId: 'comp-input-001'
    storyId: 'story-input-001'
```

The generator will create an example file on first run if it doesn't exist.

## Generated Story Format

```typescript
// Auto-generated by TraceRTM Storybook Generator
import type { Meta, StoryObj } from '@storybook/react-vite'
import { Button } from '@tracertm/ui'

const meta: Meta<typeof Button> = {
  title: 'Components/Button',
  component: Button,
  tags: ['autodocs'],
  argTypes: {
    variant: {
      control: 'select',
      options: ['default', 'destructive', 'outline', 'secondary', 'ghost', 'link'],
    },
    size: {
      control: 'select',
      options: ['default', 'sm', 'lg', 'icon'],
    },
    disabled: { control: 'boolean' },
  },
  parameters: {
    design: {
      type: 'figma',
      url: 'https://www.figma.com/file/xxx/Design-System?node-id=123',
    },
    trace: {
      componentId: 'comp-button-001',
      storyId: 'story-button-001',
    },
  },
}

export default meta
type Story = StoryObj<typeof meta>

export const Default: Story = {
  args: {
    children: 'Default',
    variant: 'default',
  },
}

export const Destructive: Story = {
  args: {
    children: 'Destructive',
    variant: 'destructive',
  },
}

// ... more variants ...

export const AllVariants: Story = {
  render: () => (
    <div className="flex flex-wrap gap-4">
      <Button variant="default">Default</Button>
      <Button variant="destructive">Destructive</Button>
      <Button variant="outline">Outline</Button>
      <Button variant="secondary">Secondary</Button>
      <Button variant="ghost">Ghost</Button>
      <Button variant="link">Link</Button>

  ),

```

## Component Analysis

The generator uses `ts-morph` to analyze TypeScript components and extract:

1. **Component Name**: From forwardRef or function component
2. **Export Type**: Default or named export
3. **CVA Variants**: Automatically detects class-variance-authority usage
4. **Props**: Extracts from TypeScript interfaces/types
5. **Default Values**: From CVA defaultVariants

## Supported Patterns

### CVA Components (Recommended)

```typescript
import { cva, type VariantProps } from 'class-variance-authority'

const buttonVariants = cva('base-classes', {
  variants: {
    variant: {
      default: 'variant-classes',
      destructive: 'destructive-classes',
    },
    size: {
      default: 'size-classes',
      sm: 'small-classes',
    },
  },
  defaultVariants: {
    variant: 'default',
    size: 'default',
  },
})

export interface ButtonProps extends VariantProps<typeof buttonVariants> {}

export const Button = forwardRef<HTMLButtonElement, ButtonProps>(...)
```

### Standard Components

```typescript
export interface InputProps extends React.InputHTMLAttributes<HTMLInputElement> {}

export const Input = forwardRef<HTMLInputElement, InputProps>(
  ({ className, type, ...props }, ref) => {
    return <input type={type} ref={ref} {...props} />
  }
)
```

## Troubleshooting

### No components found

- Check `componentsDir` paths in config
- Verify component files match `includePatterns`
- Ensure files don't match `excludePatterns`

### Import paths incorrect

- Update `importPathMapping` in config
- Ensure workspace packages are correctly named

### Variants not detected

- Verify CVA is imported and used correctly
- Check variant object structure matches expected pattern

### Design metadata not loading

- Verify YAML file exists at `designMetadataPath`
- Check YAML syntax is valid
- Run with `--verbose` for detailed error messages
