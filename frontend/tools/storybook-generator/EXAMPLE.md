# Example: Auto-Generating Stories

This example walks through creating a new component and auto-generating its Storybook story.

## Example Component

Let's say you create a new `Badge` component:

```typescript
// packages/ui/src/components/Badge.tsx
import { cva, type VariantProps } from 'class-variance-authority'
import * as React from 'react'
import { cn } from '../lib/utils'

const badgeVariants = cva(
  'inline-flex items-center rounded-full px-2.5 py-0.5 text-xs font-semibold transition-colors',
  {
    variants: {
      variant: {
        default: 'bg-primary text-primary-foreground hover:bg-primary/80',
        secondary: 'bg-secondary text-secondary-foreground hover:bg-secondary/80',
        destructive: 'bg-destructive text-destructive-foreground hover:bg-destructive/80',
        success: 'bg-green-500 text-white hover:bg-green-600',
        warning: 'bg-yellow-500 text-white hover:bg-yellow-600',
        outline: 'border border-input bg-background hover:bg-accent',
      },
    },
    defaultVariants: {
      variant: 'default',
    },
  }
)

export interface BadgeProps
  extends React.HTMLAttributes<HTMLDivElement>,
    VariantProps<typeof badgeVariants> {}

export const Badge = React.forwardRef<HTMLDivElement, BadgeProps>(
  ({ className, variant, ...props }, ref) => {
    return <div ref={ref} className={cn(badgeVariants({ variant }), className)} {...props} />
  }
)
Badge.displayName = 'Badge'
```

## What the Generator Detected

From the Badge component, the generator automatically:

1. **Detected the component name:** `Badge`
2. **Found CVA variants:**
   - Variant name: `variant`
   - Options: `['default', 'secondary', 'destructive', 'success', 'warning', 'outline']`
   - Default value: `'default'`
3. **Extracted props:** `BadgeProps` (extends HTMLDivElement attributes)
4. **Generated stories:** One for each variant option + showcase
5. **Created controls:** Select control for variant picker
6. **Added metadata:** Figma URL, component ID, story ID

## Manual Customization

After generation, you can customize stories by editing the file:

```typescript
// Add a custom story
export const WithIcon: Story = {
  render: () => (
    <Badge>
      <svg className="mr-1 h-3 w-3" /* ... */ />
      With Icon
    </Badge>
  ),
}

// Add story-specific args
export const LongText: Story = {
  args: {
    children: 'This is a very long badge text that might wrap',
    variant: 'default',
  },
}
```

Just remember: if you regenerate with `--force`, manual changes will be lost. Instead:

1. Generate base stories with the tool
2. Copy generated file to a custom name (e.g., `Badge.custom.stories.tsx`)
3. Customize the copy
4. Keep original for regeneration

## Tips for Best Results

### 1. Use CVA for Variants

```typescript
// ✅ Good - Auto-generates variant stories
const variants = cva('base', {
  variants: { variant: { primary: '...', secondary: '...' } }
})

// ❌ Not detected - Manual conditional classes
className={variant === 'primary' ? '...' : '...'}
```

### 2. Define Props Interface

```typescript
// ✅ Good - Props extracted
export interface BadgeProps extends VariantProps<typeof badgeVariants> {}

// ⚠️ Works but less info - Props not extracted
export const Badge = ({ ...props }) => ...
```

### 3. Use forwardRef Pattern

```typescript
// ✅ Good - Component detected
export const Badge = React.forwardRef<HTMLDivElement, BadgeProps>(...)

// ✅ Also good
export const Badge = forwardRef<HTMLDivElement, BadgeProps>(...)
```

### 4. Set displayName

```typescript
// ✅ Good - Better debugging
Badge.displayName = 'Badge';
```

## Common Use Cases

### Use Case 1: New Design System Component

1. Create component with CVA
2. `bun run generate:stories --component ComponentName`
3. View in Storybook
4. Add Figma URL
5. Regenerate with `--force`

### Use Case 2: Bulk Story Generation

1. Create multiple components
2. `bun run generate:stories`
3. Review all generated stories
4. Customize if needed

### Use Case 3: Update After Component Change

1. Modify component variants
2. `bun run generate:stories --component ComponentName --force`
3. Stories automatically updated with new variants

### Use Case 4: Onboarding New Developer

1. Clone repo
2. `bun install`
3. `bun run generate:stories`
4. `bun run dev:storybook`
5. All stories ready to view

From component creation to documented, tested story: **less than 1 minute**.
